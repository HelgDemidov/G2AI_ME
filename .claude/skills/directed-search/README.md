# Как запустить скилл directed-search

Канон протокола — файл `SKILL.md` рядом с этим README (обычный текстовый файл, ничего
специфичного для одного инструмента). Ниже — три способа его запустить, от простого к
более гибкому. Если вы не программист — читайте только «Способ 1», этого достаточно для
повседневной работы.

## Способ 1 — Claude Code (по умолчанию, как сейчас)

Откройте Claude Code в корне репозитория и наберите:

```
/directed-search <слаг-кампании> <ось> <коды стран>
```

Например: `/directed-search small-states-wave1 agentic_g2ai me ee is mt`. Всё — Claude Code
сам найдёт и прочитает `SKILL.md` и проведёт кампанию по протоколу.

## Способ 2 — headless-запуск через Claude Agent SDK (для будущих автоматических кампаний по расписанию)

Этот способ нужен, только когда кампании нужно запускать без открытого чата — например,
по cron'у ночью. Сейчас такого сценария нет, поэтому ниже РЕЦЕПТ (что написать, когда
понадобится), а не готовый закоммиченный скрипт — закоммиченный код без реального
использования быстро устаревает (версии SDK меняются) и никто не узнает, что он сломался,
пока не понадобится.

```python
# pip install claude-agent-sdk  (+ переменная окружения ANTHROPIC_API_KEY)
import anyio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main() -> None:
    async for msg in query(
        prompt="/directed-search small-states-wave1 agentic_g2ai me ee is mt",
        options=ClaudeAgentOptions(
            cwd="/путь/к/G2AI_ME",
            setting_sources=["project"],          # читает .claude/skills/ этого репо
            allowed_tools=["Bash", "WebSearch", "Read", "Glob", "Grep"],
            permission_mode="acceptEdits",
        ),
    ):
        print(msg)

anyio.run(main)
```

## Способ 3 — любая модель через OpenRouter (без Claude вообще)

Это для случая «я не хочу платить/пользоваться Claude, но у меня есть ключ OpenRouter».
Своего инструмента для этого мы не пишем — пользуемся готовой сторонней программой,
которая умеет то же самое (читать SKILL.md-файлы и вызывать инструменты типа поиска и
терминала). Инструкция «на пальцах», шаг за шагом:

1. **Установите Crush** — это бесплатная маленькая программа для терминала. Один из
   вариантов установки:
   - `brew install charmbracelet/tap/crush` (если у вас Homebrew, Mac/Linux)
   - `npm install -g @charmland/crush` (если у вас Node.js)
2. **Ключ OpenRouter уже есть в проекте** (файл `.env`, переменная `OPENROUTER_API_KEY`).
   При первом запуске Crush спросит, каким провайдером пользоваться — выберите OpenRouter,
   вставьте ключ, затем выберите модель посильнее (с «рассуждением» / reasoning — в списке
   моделей OpenRouter такие обычно помечены явно; берите актуальный топ-вариант).
3. **Откройте терминал ИМЕННО в корневой папке репозитория** (`G2AI_ME`) и наберите:
   ```
   crush
   ```
4. **Спросите агента: «какие скиллы тебе доступны?»**. Если в ответе есть `directed-search` —
   просто попросите: «выполни скилл directed-search, кампания <слаг>, ось <ось>,
   юрисдикции <коды>».
5. **Если скилла в списке нет** (программа не сканирует `.claude/skills/` автоматически) —
   это работает с ЛЮБЫМ агентом, который умеет читать файлы. Просто напишите ему:
   > прочти файл `.claude/skills/directed-search/SKILL.md` и выполни этот протокол;
   > кампания <слаг>, ось <ось>, юрисдикции <коды>.

   Это и есть весь секрет переносимости: канон протокола — обычный текстовый файл,
   автоматическое обнаружение скиллов — просто удобство, а не обязательное условие.

## Честная оговорка

Протокол рассчитан на СИЛЬНУЮ модель. На слабой модели ничего не «сломается» технически —
пострадает КАЧЕСТВО суждений внутри протокола: проверка издателя, оценка сниппетов
поисковой выдачи, отсев фейковых/устаревших источников (см. `SKILL.md`, раздел
«Verification before inject»). Если результаты кампании выглядят подозрительно (слишком
мало кандидатов, странные издатели, нестыковки дат) — в первую очередь проверьте, какая
модель вела кампанию.
